# 最終検証プラン: カバレッジ取得と動作確認

## 目的
実装およびセキュリティ硬化、アーキテクチャ刷新が完了したシステムの最終的なテストカバレッジを確認し、すべての機能が意図通りに動作することを検証する。

## 検証ステップ
1.  **単体テストとカバレッジの実行**:
    - `pytest --cov=skills.burnt_toast_skill tests/test_burnt_toast_skill.py --cov-report=term-missing` を実行。
2.  **結果の分析**:
    - 改善後のカバレッジ率を確認。
    - 未カバー箇所が許容範囲（例外処理の極端なケース等）であることを確認。
3.  **E2Eテストの再確認**:
    - `python3 tests/e2e_test.py` を実行し、PowerShell コマンドが正常に送信されることを再確認。

## 期待される結果
- カバレッジ率が 80% 以上であること。
- セキュリティ対策（EncodedCommand, Path Traversal）のパスがカバーされていること。
- E2Eテストにおいて、PowerShell の戻り値が 0 (成功) であること。
