# BurntToast を活用した Coding Agent の感情・イメージ伝達トースト提案

BurntToast [[5]] などの情報を基に、Windows 直・WSL から `powershell.exe` 経由で「感情やイメージを伝える」トースト通知の設計案を提案します。

---

## 🔧 前提：環境セットアップ

### Windows側（初回のみ）
```powershell
# BurntToast のインストール（管理者権限または -Scope CurrentUser）
Install-Module -Name BurntToast -Force -Scope CurrentUser
Import-Module BurntToast
```

### WSL側から呼び出すラッパースクリプト例
```bash
# ~/.local/bin/win-notify.sh
#!/bin/bash
powershell.exe -NoProfile -ExecutionPolicy Bypass -Command "Import-Module BurntToast; New-BurntToastNotification @args"
```
```bash
# 使用例
win-notify.sh -Text "完了", "タスクが終了しました" -AppLogo "/mnt/c/agents/icons/success.png"
```

> 📝 参考：WSLからWindows通知を送る実装例は複数報告されています [[18]][[20]][[21]]

## 🎭 感情・イメージ伝達の4次元アプローチ

| 次元 | 機能 | 感情表現の例 | BurntToastパラメータ |
|------|------|-------------|---------------------|
| **🎨 視覚** | 画像・レイアウト | 😊 成功＝緑アイコン、😰 エラー＝赤アイコン | `-AppLogo`, `-HeroImage`, `New-BTImage -Crop Circle` |
| **🔊 聴覚** | サウンド | ⏰ 緊急＝アラーム音、🔕 静か＝Silent | `New-BTAudio -Source ms-winsoundevent:*` |
| **🎬 シナリオ** | 表示挙動 | 📞 重要＝incomingCall、⚠️ 緊急＝Urgent | `-Scenario`, `-Urgent` |
| **🎯 インタラクション** | ボタン・進捗 | ✅ 承認＝緑ボタン、🗑️ 削除＝赤ボタン | `New-BTButton -Color Green/Red`, `New-BTProgressBar` |

## 💡 実践パターン集

### 1. ✅ 成功・完了通知（ポジティブ感情）
```powershell
$logo = New-BTImage -Source 'C:\agents\icons\success.png' -AppLogoOverride -Crop Circle
$audio = New-BTAudio -Source 'ms-winsoundevent:Notification.Mail'

New-BurntToastNotification `
    -Text '✨ タスク完了', 'コーディングが正常に終了しました' `
    -AppLogo 'C:\agents\icons\success.png' `
    -Attribution 'Coding Agent' `
    -Sound 'ms-winsoundevent:Notification.Mail' `
    -Button (New-BTButton -Content '確認' -Arguments 'task:review' -Color Green)
```

### 2. ❌ エラー・警告通知（ネガティブ感情）
```powershell
New-BurntToastNotification `
    -Text '⚠️ 実行エラー', '構文チェックに失敗しました' `
    -AppLogo 'C:\agents\icons\error.png' `
    -Sound 'ms-winsoundevent:Notification.Fault' `
    -Urgent `  # Focus Assist を突破
    -Button ( `
        (New-BTButton -Content '詳細' -Arguments 'error:details'),
        (New-BTButton -Content '再試行' -Arguments 'task:retry' -Color Green),
        (New-BTButton -Content 'スキップ' -Arguments 'task:skip' -Color Red)
    )
```

### 3. 📞 重要なユーザー確認（incomingCall シナリオ）
```powershell
# incomingCall シナリオではテキストが中央揃えになり、着信風デザインに [[3]]
$visual = New-BTVisual -BindingGeneric (
    New-BTBinding -Children (
        (New-BTText -Content '🤖 Agent の確認が必要です'),
        (New-BTText -Content 'この変更を適用しますか？')
    ) -AppLogoOverride (New-BTImage -Source 'C:\agents\icons\agent.png' -Crop Circle)
)

$content = New-BTContent -Visual $visual -Scenario IncomingCall -Audio (New-BTAudio -Source 'ms-winsoundevent:Call')

# Submit-BTNotification で送信（New-BurntToastNotification より柔軟）
Submit-BTNotification -Content $content -Actions (
    New-BTAction -Buttons (
        (New-BTButton -Content '✅ 適用' -Arguments 'confirm:yes' -Color Green),
        (New-BTButton -Content '❌ 却下' -Arguments 'confirm:no' -Color Red)
    )
)
```

### 4. 📊 進捗・待機中の感情表現（ProgressBar + 間接的安心感）
```powershell
# 進捗バーで「待たせている」ことを視覚的に伝え、不安を軽減
$progress = New-BTProgressBar -Title '処理中' -Status 'モデル推論中...' -Value 0.0 -ValueDisplay '0%'

$toast = New-BurntToastNotification `
    -Text '🔄 処理中です', 'しばらくお待ちください' `
    -ProgressBar $progress `
    -AppLogo 'C:\agents\icons\thinking.png' `
    -UniqueIdentifier 'agent-progress' `  # 同一IDで更新可能
    -Silent

# 進捗更新ループ（例）
0.0, 0.3, 0.6, 0.9, 1.0 | ForEach-Object {
    $updated = New-BTProgressBar -Status '処理中...' -Value $_ -ValueDisplay "$($_*100)%"
    Update-BTNotification -UniqueIdentifier 'agent-progress' -ProgressBar $updated
    Start-Sleep -Seconds 2
}
```

### 5. 🎨 イメージ重視の「ヒーロー画像」通知
```powershell
# 大きな画像で視覚的インパクト（例：成果物のサムネイル表示）
New-BurntToastNotification `
    -Text '🎨 生成完了', '画像が作成されました' `
    -HeroImage 'C:\agents\output\preview.png' `
    -Attribution 'Generated by Agent' `
    -Button (New-BTButton -Content '開く' -Arguments 'file:C:\agents\output\preview.png')
```

## ⚙️ WSL連携の実装上の注意点

1. **パス変換**: WSLのパス（`/mnt/c/...`）をWindows形式（`C:\...`）に変換する必要があります
   ```bash
   # WSL側で変換関数
   winpath() { echo "$(wslpath -w "$1")"; }
   ```

2. **画像サイズ制限**: 遠隔画像は 3MB 以下、メーター接続時は 1MB 以下 [[3]]。ローカルファイル推奨。

3. **AppUserModelID**: 通知の「発信元」表示をカスタマイズしたい場合は、`New-BTShortcut` で専用ショートカットを作成し、そこから PowerShell を起動するとブランド表示が可能 [[5]]。

4. **Focus Assist 対策**: 重要な通知には `-Urgent` [[14]]、または `Scenario: Urgent` [[3]] を使用。

## 🚀 推奨アーキテクチャ

```
[ Coding Agent (WSL) ]
         │
         ▼
[ 感情テンプレート選択 ] → [ PowerShell コマンド生成 ]
         │
         ▼
[ powershell.exe 経由で実行 ]
         │
         ▼
[ BurntToast → Windows Toast ]
         │
         ▼
[ ユーザーがボタン押下 ] → [ 戻り値を Agent へフィードバック ]
```

ボタン押下の処理は `-ActivatedAction { ... }` [[14]] や引数解析で実現可能です。

## ✅ まとめ：感情伝達のポイント

| 伝えたい感情 | 推奨組み合わせ |
|-------------|---------------|
| 😊 安心・完了 | 緑アイコン + 軽快な音 + 緑ボタン |
| 😰 警告・エラー | 赤アイコン + 警告音 + `-Urgent` + 赤ボタン |
| 🤔 待機・処理中 | 進捗バー + 思考アイコン + 無音 |
| 📞 重要確認 | `IncomingCall` + 着信音 + 中央揃えテキスト |
| 🎨 視覚的インパクト | `HeroImage` + 大きなサムネイル |

BurntToast は PowerShell 単体でリッチなトーストを構築できるため、Coding Agent の「状態」や「感情」を直感的に伝える強力な手段になります。特に `-Urgent` [[14]]、`Scenario` [[3]]、`-Color` [[14]] などを組み合わせることで、単なる通知を超えた「対話的な体験」を設計可能です。

必要に応じて、さらに具体的な実装コードやテンプレートの拡張もサポートできます。

## 🐍 burnt_toast_skill.py
```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
BurntToast Notification Skill
- 感情テンプレートに基づき Windows トースト通知を生成・送信
- WSL/Windows 両対応、powershell.exe 経由で実行
- 会話コンテキストから即時呼び出し可能
"""

import json
import subprocess
import shutil
from pathlib import Path
from typing import Optional, List, Dict, Union
from dataclasses import dataclass, asdict


@dataclass
class ToastButton:
    label: str
    arguments: str
    color: Optional[str] = None  # Green, Red, Yellow 等

    def to_ps(self) -> str:
        color_part = f' -Color {self.color}' if self.color else ''
        return f"(New-BTButton -Content '{self.label}' -Arguments '{self.arguments}'{color_part})"


@dataclass
class ToastRequest:
    emotion: str
    title: str
    message: str
    unique_id: Optional[str] = None
    progress_value: Optional[float] = None  # 0.0 ~ 1.0
    progress_status: Optional[str] = None
    hero_image: Optional[str] = None
    custom_buttons: Optional[List[ToastButton]] = None
    override_sound: Optional[str] = None
    attribution: str = "Coding Agent"


class BurntToastSkill:
    """BurntToast を使った感情伝達トーストの生成・送信スキル"""

    def __init__(self, config_path: Optional[str] = None):
        self.config = self._load_config(config_path)
        self.ps_exe = self.config.get("powershell_exe", "powershell.exe")
        self.ps_args = self.config.get("powershell_args", [])
        self.icon_base = Path(self.config.get("icon_base_path", ""))
        self.default_app_id = self.config.get("default_app_id")
        self.templates = self.config.get("emotion_templates", {})

    def _load_config(self, path: Optional[str]) -> dict:
        if path and Path(path).exists():
            with open(path, 'r', encoding='utf-8') as f:
                return json.load(f)
        # デフォルト設定（最小限）
        return {
            "powershell_exe": "powershell.exe",
            "powershell_args": ["-NoProfile", "-ExecutionPolicy", "Bypass"],
            "emotion_templates": {}
        }

    def _wsl_to_win_path(self, path: str) -> str:
        """WSL パスを Windows パスに変換（必要時）"""
        if not path.startswith('/mnt/'):
            return path
        try:
            result = subprocess.run(
                ['wslpath', '-w', path],
                capture_output=True, text=True, check=True
            )
            return result.stdout.strip()
        except (subprocess.CalledProcessError, FileNotFoundError):
            # 変換失敗時はそのまま返す（Windows 側で処理）
            return path.replace('/mnt/c/', 'C:\\').replace('/', '\\')

    def _get_icon_path(self, icon_name: str) -> str:
        full_path = self.icon_base / icon_name if self.icon_base else Path(icon_name)
        return self._wsl_to_win_path(str(full_path))

    def _build_ps_command(self, req: ToastRequest, template: dict) -> str:
        """PowerShell コマンド文字列を構築"""
        prefix = template.get("text_prefix", "")
        title = f"{prefix} {req.title}".strip()
        
        # 基本パラメータ
        params = [
            f"-Text '{title}'",
            f"-Text '{req.message}'",
            f"-Attribution '{req.attribution}'"
        ]

        # 画像設定
        icon = template.get("icon")
        if icon:
            icon_path = self._get_icon_path(icon)
            params.append(f"-AppLogo '{icon_path}'")
        
        if req.hero_image:
            hero_path = self._wsl_to_win_path(req.hero_image)
            params.append(f"-HeroImage '{hero_path}'")

        # 音声設定
        sound = req.override_sound or template.get("sound")
        if sound:
            params.append(f"-Sound 'ms-winsoundevent:{sound}'")
        elif template.get("silent"):
            params.append("-Silent")

        # 緊急・シナリオ
        if template.get("urgent"):
            params.append("-Urgent")
        scenario = template.get("scenario")
        if scenario:
            params.append(f"-Scenario '{scenario}'")

        # 進捗バー
        if template.get("use_progress") or req.progress_value is not None:
            value = req.progress_value if req.progress_value is not None else 0.0
            status = req.progress_status or template.get("progress_title", "処理中")
            display = f"{int(value * 100)}%"
            params.append(f"-ProgressBar (New-BTProgressBar -Title '{status}' -Value {value} -ValueDisplay '{display}')")

        # ユニークID（更新用）
        if req.unique_id:
            params.append(f"-UniqueIdentifier '{req.unique_id}'")

        # ボタン設定
        buttons = req.custom_buttons
        if not buttons and template.get("button_label"):
            # テンプレートから単一ボタン生成
            btn = ToastButton(
                label=template["button_label"],
                arguments=f"action:{req.emotion}",
                color=template.get("button_color")
            )
            buttons = [btn]
        
        if buttons:
            btn_strs = [btn.to_ps() for btn in buttons]
            if len(btn_strs) == 1:
                params.append(f"-Button {btn_strs[0]}")
            else:
                joined = ", ".join(btn_strs)
                params.append(f"-Button ({joined})")

        # PowerShell コマンド全体
        ps_cmd = f"Import-Module BurntToast -ErrorAction SilentlyContinue; New-BurntToastNotification {' '.join(params)}"
        return ps_cmd

    def notify(self, req: ToastRequest) -> bool:
        """トースト通知を送信"""
        template = self.templates.get(req.emotion, {})
        ps_command = self._build_ps_command(req, template)
        
        full_cmd = [self.ps_exe] + self.ps_args + ["-Command", ps_command]
        
        try:
            # WSL 環境でも powershell.exe が PATH 上にあることを想定
            result = subprocess.run(
                full_cmd,
                capture_output=True,
                text=True,
                timeout=10,
                shell=(shutil.which(self.ps_exe) is None)  # WSL 用のフォールバック
            )
            if result.returncode != 0:
                print(f"[BurntToast] Error: {result.stderr.strip()}")
                return False
            return True
        except Exception as e:
            print(f"[BurntToast] Exception: {e}")
            return False

    # ========== 簡易呼び出しメソッド（会話中にすぐ使える） ==========
    
    def send(self, emotion: str, title: str, message: str, **kwargs) -> bool:
        """最小引数で通知を送信（会話フロー向け）"""
        req = ToastRequest(emotion=emotion, title=title, message=message, **kwargs)
        return self.notify(req)
    
    def success(self, title: str, message: str, **kwargs) -> bool:
        return self.send("success", title, message, **kwargs)
    
    def error(self, title: str, message: str, **kwargs) -> bool:
        return self.send("error", title, message, **kwargs)
    
    def warning(self, title: str, message: str, **kwargs) -> bool:
        return self.send("warning", title, message, **kwargs)
    
    def waiting(self, title: str, message: str, 
                unique_id: str, value: float = None, status: str = None, **kwargs) -> bool:
        """進捗通知（更新可能）"""
        req = ToastRequest(
            emotion="waiting", title=title, message=message,
            unique_id=unique_id, progress_value=value, progress_status=status, **kwargs
        )
        return self.notify(req)
    
    def confirm(self, title: str, message: str, 
                on_yes: str = "confirm:yes", on_no: str = "confirm:no", **kwargs) -> bool:
        """確認ダイアログ風トースト"""
        buttons = [
            ToastButton("✅ 適用", on_yes, "Green"),
            ToastButton("❌ 却下", on_no, "Red")
        ]
        req = ToastRequest(
            emotion="confirmation", title=title, message=message,
            custom_buttons=buttons, **kwargs
        )
        return self.notify(req)
    
    def update_progress(self, unique_id: str, value: float, status: Optional[str] = None) -> bool:
        """進捗バーを更新（同一ユニークIDで上書き）"""
        # 進捗更新は ProgressBar 付きで再送信することで実現
        req = ToastRequest(
            emotion="waiting", title="", message="",
            unique_id=unique_id, progress_value=value, progress_status=status
        )
        template = self.templates.get("waiting", {})
        ps_command = self._build_ps_command(req, template)
        full_cmd = [self.ps_exe] + self.ps_args + ["-Command", ps_command]
        try:
            subprocess.run(full_cmd, capture_output=True, text=True, timeout=5)
            return True
        except:
            return False


# ========== Skill 登録用エントリーポイント ==========

def get_skill():
    """nanobot / Agent フレームワーク向け skill 取得関数"""
    skill = BurntToastSkill()
    
    # 会話中に呼び出せるコマンド名を登録（例）
    commands = {
        "notify": skill.send,
        "toast_success": skill.success,
        "toast_error": skill.error,
        "toast_warning": skill.warning,
        "toast_waiting": skill.waiting,
        "toast_confirm": skill.confirm,
        "toast_update": skill.update_progress,
    }
    
    return {
        "name": "burnt_toast",
        "description": "感情・イメージを伝える Windows トースト通知を送信するスキル",
        "commands": commands,
        "instance": skill
    }


# ========== 直接実行時のテスト用 ==========
if __name__ == "__main__":
    import sys
    
    skill = BurntToastSkill()
    
    # 簡易テスト
    if len(sys.argv) < 4:
        print("Usage: python burnt_toast_skill.py <emotion> <title> <message>")
        print("Emotions: success, error, warning, waiting, confirmation, info")
        sys.exit(1)
    
    emotion, title, message = sys.argv[1], sys.argv[2], " ".join(sys.argv[3:])
    result = skill.send(emotion, title, message)
    print(f"Notification sent: {result}")
```

## 📖 概要

**BurntToast Notification Skill** は、Coding Agent がユーザーに対して「単なるテキスト通知」を超えて、**感情・状態・緊急性**を直感的に伝えるための Windows トースト通知スキルです。

[Windos/BurntToast](https://github.com/Windos/BurntToast) PowerShell モジュールをバックエンドに使用し、以下のことを実現します：

```
✅ 処理完了を「✨ 緑アイコン + 軽快な音」で祝福
❌ エラーを「⚠️ 赤アイコン + 警告音 + 緊急フラグ」で強調
🔄 待機中を「進捗バー + 思考アイコン」で不安を軽減
🤖 確認事項を「着信風デザイン + 緑/赤ボタン」で対話的に
```

会話型エージェント（nanobot, Qwen, etc.）と連携し、**会話コンテキストから即時呼び出し可能**な設計を採用しています。
