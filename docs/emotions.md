# 🎭 感情表現ガイド

BurntToast Notification Skill は、エージェントの「状態」や「気持ち」を視覚・聴覚を通じて直感的に伝えるための仕組みを提供します。

---

## 🎭 感情テンプレートのマッピング

デフォルトでは `skills/burnt_toast_config.json` にて以下の 6 つの感情テンプレートが定義されています。

| 感情 | 絵文字 | アイコン例 | サウンド | 挙動の特徴 |
| :--- | :---: | :--- | :--- | :--- |
| **Success** | ✨ | 🟢 緑チェック | Mail | 安心感を与える軽快な音 |
| **Error** | ⚠️ | 🔴 警告 | Default | **Urgent** フラグで強調。集中モードを突破 |
| **Warning** | 🔶 | 🟡 警告 | Reminder | 注意を促す。緊急ではない |
| **Waiting** | 🔄 | 🔵 思考 | Silent | 無音。進捗バーで安心感を提供 |
| **Confirm** | 🤖 | 🟣 エージェント | Call | 着信風のデザイン。ユーザーの判断を待つ |
| **Info** | 💡 | 🔵 情報 | Default | 標準的なお知らせ |

---

## 🎨 表現を豊かにする要素

### 1. 視覚的接頭辞 (Text Prefix)
通知タイトルの先頭に絵文字を付与します。これにより、通知がポップアップした瞬間に「良い知らせ」か「悪い知らせ」かを判別できます。

### 2. ヒーロー画像 (Hero Image)
大きな画像を通知の上部に表示します。生成した画像（AIアートやグラフなど）をユーザーに即座に見せたい場合に有効です。

### 3. シナリオ (Scenario)
- **Urgent**: 他の通知よりも優先的に表示され、ユーザーが確認するまで画面に残りやすくなります。
- **IncomingCall**: 通話着信のような特別なレイアウトになり、タイトルが中央に大きく表示されます。

---

## ⚙️ カスタマイズ方法

新しい感情や、特定のプロジェクト向けの通知スタイルを追加するには、`burnt_toast_config.json` を編集します。

```json
"celebration": {
  "text_prefix": "🎊",
  "icon": "celebrate.png",
  "sound": "Alarm",
  "button_color": "Green",
  "button_label": "シェアする"
}
```

### 利用可能なサウンド
Windows の標準サウンドイベント名を指定可能です：
`Default`, `IM`, `Mail`, `Reminder`, `SMS`, `Alarm`, `Call` (1〜10)

---

## 🧠 エージェントへの指示 (Prompting)

エージェントがこのスキルを使う際のガイドライン：
- **成功時**: `success` メソッドを使い、✨ などの明るい表現をタイトルに含めてください。
- **待機時**: `waiting` を使い、進捗バーを更新することで、ユーザーが「ハングした」と誤解するのを防いでください。
- **確認時**: `confirm` を使い、ユーザーに「適用」か「却下」かの明確な選択肢を提示してください。
